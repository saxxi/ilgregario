## Deploy su Render.com — Checklist

### 1. Git repo
- [ ] `git init` + primo commit
- [ ] Push su GitHub (repo privato)

### 2. Supabase remoto
- [ ] Applicare migration `20260427000001_init.sql` sul DB remoto  
  ```
  supabase db push --linked
  ```
  oppure incollare la migration direttamente su Supabase Studio > SQL Editor.
- [ ] **`SUPABASE_KEY`**: usare la **service role key** (non la anon key) — si trova in Supabase Dashboard > Project Settings > API.

### 3. Render.com
- [ ] Creare nuovo Web Service, puntare al repo GitHub
- [ ] Build command: `pip install uv && uv sync --frozen`
- [ ] Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
- [ ] Aggiungere env vars nel Render dashboard:

| Variabile | Valore |
|---|---|
| `ADMIN_USERNAME` | `admin` (o scegli) |
| `ADMIN_PASSWORD` | scegli password sicura |
| `SUPABASE_URL` | URL progetto Supabase (es. `https://xyz.supabase.co`) |
| `SUPABASE_KEY` | **service role key** (tienila segreta) |
| `SECRET_KEY` | stringa random lunga (es. `openssl rand -hex 32`) |
| `OPENAI_API_KEY` | opzionale — solo se usi il fallback parsing |

### 4. `.env` locale vs produzione
- `.env` → solo locale, non committare mai
- Aggiungere `.env` a `.gitignore` prima del primo commit
